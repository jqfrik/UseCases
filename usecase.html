Пример
PrimaryActor(Действующие лица) Зарегистрированный пользователь
Target(Цель) Купить базовый тариф
Trigger(Действие пользователя) Пользователь перешёл в раздел подписки
Preconditions(Предусловия)
Pre1.Пользователь аутентифицирован
Pre2.Пользователь просматривает страницу с тарифами
Pre3.Пользователь выбирает необходимые параметры тарифа и способ оплаты
Pre4.Пользователь покупает тариф
Postconditions(Постусловия) POST1.Система открывает пользователю доступ к системе,с привилегиями,доступными в рамках тарифа
Normal Flow:(Обычный правильный поток действий)
1.Система приняла информацию о доступных пользователю привилегиях и сроках их использования,согласно купленному тарифу
2.Система проверила,что на данный момент у пользователя нет действующего тарифа
3.Если оплата была проведена успешно через платёжную систему ,то автоматически открывается доступ пользователя к системе,согласно параметрам тарифа
4.Если оплата была проведена успешно через счет,то тариф подключается вручную
5.Система отправила сообщение об активации подписки на email пользователя

PrimaryActor(Действующие лица) Незарегистрованный пользователь
Target(Цель) Авторизоваться в системе
Trigger(Действие пользователя) Пользователь открывает exe файл
Preconditions(Предусловия) --------------
Postconditions(Постусловия) POST1.Система открывает пользователю доступ к системе,с привилегиями,доступными в рамках текущего пользователя
Normal Flow:(Обычный правильный поток действий)
Пользователь находится в окне ввода авторизационных данным
Пользователь ввёл авторизационные данные
Система проверила данные о текущем пользователе и успешно нашла пользователя в БД
Система отправила ключ доступ для доступа к данным системы
Пользователь переходит в главную панель управления программой
PrimaryActor(Действующие лица) Зарегистрированный пользователь
Target(Цель) Запуск программы
Trigger(Действие пользователя) Пользователь нажимает кнопку Старт
Preconditions(Предусловия)
Пользователь успешно авторизован
Пользователь имеет доступное время для работы с программой
Заранее установленные настройки программы
Postconditions(Постусловия)
Открываются окна браузеров в которых пользователь непосредственно может наблюдать работу программы
Система записывает действия программы в журналы(далее "Логи"),которые пользователь может наблюдать
Normal Flow:(Обычный правильный поток действий)
Пользователь находится в панеле управления программой
Пользователь нажимает кнопку
Появляются окна браузеров со страницами ранее выбранных в настройках букмекеров
Действия программы записываются в журналы логирования
